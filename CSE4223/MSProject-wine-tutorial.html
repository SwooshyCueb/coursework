Getting Microsoft Project 2010 in Wine

Tested with Microsoft Project 2010 Professional SP1 32-bit and wine 1.7.20 (ors build). You will need winetricks or winetricks-amd64.
You WILL need a key from OnTheHub/MSDN/DreamSpark/wherever. You don't have to use the installer from OnTheHub/DreamSpark, but make sure you have the installer that matches your key.
You will need an ntlm_auth binary installed. This is part of Samba 3. Many distributions have it in a winbind package.
The 64-bit version of Microsoft Project does not work in wine as of this writing. We will use a 32-bit wineprefix.

Preparing the wineprefix

Pop open a terminal and run the following command. It will setup the wineprefix and open the wineprefix's registry editor.
WINEARCH=win32 WINEPREFIX=~/.msproject wine regedit
Navigate to HKEY_CURRENT_USER\Control Panel\International\ and change sList to a semicolon.
You can close the registry editor. Back in the terminal, run this command:
WINEARCH=win32 WINEPREFIX=~/.msproject winetricks msxml3
Depending on the version of winetricks and the avaliability of the installer on Microsoft's servers, you may be asked to download a file and place it in a particular folder before running the command again. You will be directed to a proper place to download the file. Once you've got the file, see the installer through.
Back in the terminal, run this command:
WINEARCH=win32 WINEPREFIX=~/.msproject winetricks ie8
This will download and run the installer for Internet Explorer 8. Again, depending on the availability of an installer on Microsoft's servers, you may be asked to download a file. It will ask you if you want to download and instuall updates. I recommend you don't. It will ask you to restart your computer. You can click either button; it doesn't matter. You don't actually need to restart.

Secure Download Manager

In order to download the installer image from OnTheHub/DreamSpark, you will need to install Secure Download Manager in wine. If you have already done this, or do not need to do this, skip to the next section.
OnTheHub/DreamSpark will likely try to give you a pkg file (likely the Mac installer) for SDM, but you need the msi. As of this writing, the msi for the english installer can be found here.
Once you have the installer downloaded, cd to its directory and run the following command.
WINEARCH=win32 WINEPREFIX=~/.msproject wine msiexec /i SDM_EN.msi
See the installer through.
Run the following command to launch SDM with your Microsoft Project 2010 order information.
WINEARCH=win32 WINEPREFIX=~/.msproject wine "C:\\users\\$USER\\Local Settings\\Application Data\\e-academy Inc\\SecureDownloadManager\\SecureDownloadManager.exe" msproj2010.sdx
where msproj2010.sdx is the sdx file provided by OnTheHub/DreamSpark.
You will probably want to change the download location to something in the wineprefix's "My Documents". Click Start Download. Once this is done, you can close SDM, but remember where you downloaded your files. Do not click launch.

Preparing the wineprefix, part 2

Back in the terminal, run this command:
WINEARCH=win32 WINEPREFIX=~/.msproject winetricks msxml6 riched30
Depending on the version of winetricks and the avaliability of the installer on Microsoft's servers, you may be asked to download a file and place it in a particular folder before running the command again. You will be directed to a proper place to download the file. Once you've got the file, see the installer through.
Now run this command:
WINEARCH=win32 WINEPREFIX=~/.msproject wine winecfg
This will open wine's configuration utility. In the libraries tab, add riched20 to the overrides list, if it's not already there. Load order should be native then builtin.
Click OK after you have ensured this override exists.

Microsoft Project

If you have an ISO, you do not need to burn it or mount it. You can simply extract it. If your ISO or installer is multiarchitecture (such as the one provided by OnTheHub/DreamSpark), we are only interested in the contents of the x86 folder.
cd into the installer foler. If the installer folder is multiarchitecture, cd into the x86 folder. Then run this command.
WINEARCH=win32 WINEPREFIX=~/.msproject wine setup.exe
Follow the installation through to completion.
You should now have Microsoft Project installed. Use the following command to launch it.
WINEARCH=win32 WINEPREFIX=~/.msproject wine ~/.msproject/drive_c/Program\ Files/Microsoft\ Office/Office14/WINPROJ.EXE
Adjust paths and such as needed.
